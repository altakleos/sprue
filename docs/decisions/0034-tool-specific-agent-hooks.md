# ADR-0034: Tool-Specific Agent Hook Files

## Context

Sprue instances are operated by LLM coding agents (Claude Code, Cursor, Kiro, Copilot, Windsurf, Cline, Roo Code, JetBrains AI, and others). The boot chain requires the agent to read `AGENTS.md` at the instance root, which points to `.sprue/engine.md` and the protocol library.

In practice, not all tools auto-discover `AGENTS.md`. Claude Code reads `CLAUDE.md`. Kiro uses `.kiro/steering/`. Cursor uses `.cursor/rules/`. A fresh session in a tool that doesn't auto-read `AGENTS.md` produces a broken boot chain — the agent doesn't recognize Sprue commands.

The AGENTS.md open spec (Linux Foundation, Dec 2025) is gaining adoption — Codex, Copilot, Cursor, Windsurf, Cline, Roo Code, and others now read it natively. But Claude Code and tool-specific config directories remain parallel paths that need bridging.

## Decision

`sprue init` generates thin shim files for all major LLM coding tools alongside `AGENTS.md`. Each shim points to `AGENTS.md` as the single source of truth — no content is duplicated.

Files generated:

| File | Tool | Content |
|---|---|---|
| `AGENTS.md` | Universal (Codex, Copilot, Cursor, Windsurf, Cline, Roo Code, Devin, Amp) | Full boot document (existing) |
| `CLAUDE.md` | Claude Code | `See @AGENTS.md` (native import directive) |
| `.kiro/steering/sprue.md` | Kiro CLI | Prose pointer to AGENTS.md |
| `.cursor/rules/sprue.mdc` | Cursor | MDC frontmatter + prose pointer |
| `.github/copilot-instructions.md` | GitHub Copilot | Prose pointer |
| `.windsurf/rules/sprue.md` | Windsurf | YAML frontmatter + prose pointer |
| `.clinerules/sprue.md` | Cline | Prose pointer |
| `.roo/rules/sprue.md` | Roo Code | Prose pointer |
| `.aiassistant/rules/sprue.md` | JetBrains AI | Prose pointer |

All shims are generated by default. No flag required. The cost of unused shim files is near zero (1–3 lines each); the cost of a broken boot chain is a confused user on first session.

Tool-specific directories (`.kiro/`, `.cursor/`, `.github/`, etc.) are committed to version control so collaborators get the same boot experience.

## Alternatives Considered

- **Generate only AGENTS.md** — rejected because Claude Code (the most popular agentic coding tool) doesn't read it. A fresh Claude Code session in a Sprue instance would fail to boot.
- **Duplicate AGENTS.md content into each tool file** — rejected because it creates N copies that drift. AGENTS.md is the single source of truth; shims point to it.
- **Flag-gated generation (`--hooks kiro,cursor`)** — rejected because it requires the user to know which tool they'll use at init time. Generating all is simpler and the files are tiny.
- **Auto-detect tool from environment** — rejected because the KB may be used by different collaborators with different tools. Generating all covers everyone.

## Consequences

Every major LLM coding tool discovers the Sprue engine on first session without user intervention. `AGENTS.md` remains the canonical boot document. Tool-specific files are 1–3 line shims that never need updating unless the tool's convention changes. The `.gitignore` template must not ignore tool directories.

Claude Code's `@AGENTS.md` import is the only true file-inclusion mechanism; all other shims use prose instructions ("Read and follow AGENTS.md") which work because these are agentic tools with file-reading capabilities.

Future tools that adopt the AGENTS.md spec need no shim — they work automatically.
