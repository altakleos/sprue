# ADR-0020: Mermaid Diagram Standard

## Context
The KB needs visual documentation for architecture, workflows, and relationships. Binary image formats (PNG, SVG exports) are not diffable in version control and cannot be read or generated by LLM agents. The platform needed a text-based diagram format.

## Decision
All diagrams use Mermaid syntax, rendered in fenced code blocks within markdown files. Mermaid is version-controllable, diffable, and both human- and LLM-readable, making it the natural fit for a git-based, agent-operated platform.

## Alternatives Considered
- **PlantUML** — requires a Java runtime for rendering; heavier dependency than Mermaid's browser-native support
- **ASCII art diagrams** — no standardized syntax; difficult to maintain and limited in expressiveness

## Consequences
Diagrams live alongside prose in markdown, staying in sync with content. LLM agents can generate and modify diagrams directly. GitHub and most markdown renderers support Mermaid natively. Complex diagrams with many nodes can become hard to read in source form.
